Editor-maintained agent instructions, rendered with the same recordlist table pattern as TaskTable.html so the heading, collapse toggle, and persistence-via-BE-user-uc all work out of the box (handled by @typo3/backend/recordlist.js, imported in Chat/Index.html). The persistent collapsed state is stored under BE_USER->uc['moduleData']['web_list']['collapsedTables']['agent-instructions']. "Always" instructions are injected into every system prompt; "on-demand" instructions are loaded by the agent via the GetInstruction tool when relevant. Editors with modify rights on tx_agent_instruction see an edit button per row plus a "+ New instruction" action in the heading; both open FormEngine (record_edit) with a returnUrl back to the AI Tasks module. Without that right, the action column is dropped entirely. Empty state: no instructions and the user may create them → just a hint plus the "+ New instruction" button, no empty table shell. No instructions and no edit rights → nothing renders. Instruction body is rich text from the RTE; rendered via f:transform.html (resolves t3:// links) piped through f:sanitize.html (XSS-safe in backend context).
